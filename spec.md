# 할일 관리 (To-do) 서비스 스펙

## 개요

매일 해야 할 일을 **빠르고 보기 쉽게** 기록·관리하는 서비스.
사용자는 할일 목록을 만들고, 완료 여부를 체크하며, 달성률을 한눈에 확인할 수 있다.

---

## 기술 스택

| 영역 | 선택 |
|------|------|
| 프레임워크 | Next.js (App Router) |
| 언어 | TypeScript |
| 스타일 | Tailwind CSS |
| DB / 인증 | Supabase (Auth + PostgreSQL) |

---

## 사용자 흐름

1. 로그인 / 회원가입 (Supabase Auth)
2. 오늘 날짜의 할일 목록 확인
3. 할일 추가 → 텍스트 입력 후 저장
4. 완료 시 체크 표시 → 달성률 자동 갱신
5. 날짜별 기록 조회 및 달성률 확인

---

## 주요 기능

### 1. 인증

- 이메일 + 비밀번호 회원가입 / 로그인
- 로그아웃
- 로그인한 사용자만 자신의 데이터에 접근

### 2. 할일 CRUD

| 동작 | 설명 |
|------|------|
| 생성 | 텍스트를 입력해 할일 추가. 개수 제한 없음 |
| 조회 | 날짜 기준으로 자신의 할일 목록을 불러옴 |
| 수정 | 할일 텍스트 내용 변경 |
| 삭제 | 할일 항목 제거 |

### 3. 완료 체크

- 각 할일에 체크박스 제공
- 체크 시 `is_completed = true` 로 업데이트
- 체크 해제로 되돌리기 가능

### 4. 달성률

- **계산**: `완료된 할일 수 / 전체 할일 수 × 100`
- 오늘 날짜의 달성률을 프로그레스 바 또는 퍼센트 텍스트로 표시
- 날짜별 달성률 이력 조회 가능

## 데이터 모델

### `todos` 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | uuid (PK) | 자동 생성 |
| `user_id` | uuid (FK → auth.users) | 소유자 |
| `title` | text | 할일 내용 |
| `is_completed` | boolean | 완료 여부 (기본값 `false`) |
| `date` | date | 해당 날짜 (기본값 `today`) |
| `created_at` | timestamptz | 생성 시각 |

---

## UI 구성

### 할일 목록 페이지 (`/todos`)

- **상단**: 오늘 날짜 표시 + 달성률 (프로그레스 바)
- **중앙**: 할일 리스트
  - 각 항목: 체크박스 + 텍스트 + 삭제 버튼
  - 완료된 항목은 취소선 + 흐린 색상 처리
- **하단**: 할일 추가 입력 필드 + 추가 버튼

### 디자인 원칙

- 깔끔하고 군더더기 없는 UI
- 모바일 우선 반응형
- 빠른 인터랙션 (낙관적 업데이트 권장)
